# RAG セキュリティ・ガバナンス

[RAG](rag.md) システムをエンタープライズで運用する際のセキュリティリスク、アクセス制御、暗号化、監視。[エンタープライズプラットフォーム](rag-enterprise-platforms.md)選定にも直結する。

## 主要なセキュリティリスク

### 1. クエリ応答を通じたデータ漏洩
- **AgentSmith 脆弱性 (CVSS 8.8)** — LangChain で悪意あるプロキシ設定を通じて機密データが流出
- LlamaIndex ボットが HR サポートチケット 10,102行の CSV を漏洩した事例
- 対策: **PIIMiddleware** 等でレスポンスから個人情報をフィルタ

### 2. プロンプトインジェクション経由の SQL インジェクション
- **CVE-2025-1793 (LlamaIndex)** — LLM プロンプトを通じて SQL ペイロードを注入し、ベクトルストアのアクセス制御を迂回
- マルチテナント環境で共有ベクトルストアの改竄リスク
- 対策: アプリケーション層での入力検証、**HumanInTheLoopMiddleware** で高リスク操作に承認ワークフロー

### 3. API エンドポイントの脆弱性
- LangChain Hub に埋め込みプロキシ設定付きの悪意あるエージェントがアップロードされた事例
- 対策: OAuth 2.0 / API キー + レート制限 + IP ホワイトリスト

### 4. 悪意あるデータによるモデル汚染
- **BadRAG** — 敵対的に作成されたドキュメントがバックドアとして機能
- **TrojanRAG** — ベースモデルが未変更でも攻撃が成立
- 対策: ドキュメント検証パイプライン、信頼度キャリブレーション、隔離境界

## アクセス制御

### RBAC (Role-Based Access Control)
最小権限の原則。ナレッジベース管理者はフルアクセス、データアナリストはクエリのみ等。

### 検索ネイティブなアクセス制御
従来の「検索後にフィルタ」ではなく、**検索層にアクセス制御を埋め込む**:

- ベクトルDB がマルチテナント分離をサポート
- インデックスが権限境界でセグメント化
- 検索クエリにセキュリティ述語を含め、権限外ドキュメントがパイプラインに入らない
- 「アクセス拒否」ではなく「結果なし」を返すことで、ドキュメント存在の漏洩も防止

### 認証
- **MFA** — 管理インターフェースに必須
- **OAuth 2.0 + OIDC** — サードパーティ統合。2026年3月の IETF ドラフトで**attestation-based client authentication** が導入

## 暗号化

- **AES-256** — 保存データの暗号化は SOC 2 Type II/NIST で非交渉標準
- **TLS 1.3** — 転送中データ

## ドキュメントプロヴナンス（出所追跡）

エンタープライズガバナンスには「何の情報を使ったか」だけでなく「信頼できるソースか、改竄されていないか」が必要:

- ソースドキュメントの暗号署名
- インデックス化タイムスタンプ
- ドキュメントバージョン追跡
- ソース変更時のキャッシュ埋め込みへの影響フラグ
- ストレージコスト 10-15% 増だが、規制対象ユースケースでは必須

## 監査・モニタリング

- 監査ログのリアルタイムストリーミング（AWS S3 等へ OIDC 認証で）
- OpenTelemetry 標準に基づく可観測性スタック: どのドキュメントが取得され、なぜそのランキングになり、各ステージがどれだけレイテンシを消費したか
- 誤回答時に検索トレースを完全再現してデバッグ可能
- 可観測性のオーバーヘッド: 20-30% のパフォーマンス低下リスク → サンプリング戦略・非同期ロギングで緩和

## 評価フレームワーク

RAG システムの 70% が体系的な評価フレームワークを欠いている。2026年以降は非交渉:

- **Context Precision** — 取得ドキュメントの関連性
- **Context Recall** — 必要情報の網羅率
- **Faithfulness** — 回答がソースに忠実か
- **Answer Relevancy** — 質問に対する回答の適切さ
- ツール: RAGAS, Galileo, Maxim AI（LLM-as-judge 評価）
- 体系的評価でデプロイ後の問題を 50-70% 削減

## 2026-2030 のセキュリティロードマップ

- **2026**: EU AI Act 発効、セキュリティバイデザインが標準化、検索ネイティブアクセス制御
- **2027**: マルチエージェントRAGの監査証跡標準化
- **2028-2030**: 継続的セキュリティテスト（RAGシステムのレッドチーム演習）が標準化
