# RAG テクノロジースタック

[RAG](rag.md) パイプラインを構成する具体的な技術要素。ベクトルDB、オーケストレーションフレームワーク、埋め込みモデルの3層。

## ベクトルデータベース

ベクトル（埋め込み）を格納し、類似度検索を行うストレージ層。選択は用途・規模・既存インフラで決まる。

### 主要6つ（大半のユースケースをカバー）

| DB | タイプ | 適用 | 強み | 弱み |
|---|---|---|---|---|
| **Pinecone** | フルマネージド・サーバーレス | 10M-100M+ ベクトル、運用ゼロ | 最も簡単、マルチリージョン | 従量課金が高くなりうる |
| **Milvus** | OSS | 数十億規模、セルフホスト | 最も人気のOSS (35K+ GitHub stars) | 運用が複雑 |
| **Weaviate** | OSS + マネージド | RAG <50M、ハイブリッド検索 | ハイブリッド検索が最良 | 14日間トライアル制限 |
| **Qdrant** | OSS + マネージド | <50M、フィルタリング | 最良の無料枠 (1GB永久)、Rust製で高速 | 10M超でスループット低下 |
| **pgvector** | PostgreSQL拡張 | 既存 PostgreSQL ユーザー | 統一データモデル、追加インフラ不要 | 100M超は非推奨 |
| **Elasticsearch** | 検索+ベクトル | 既存 Elastic ユーザー | 信頼性、エコシステム | レイテンシがやや高い |

### 専門用途

| DB | 適用 |
|---|---|
| **ChromaDB** | プロトタイプ、<10M、最良のDX |
| **Redis** | サブミリ秒レイテンシ、<10M（メモリバウンド） |
| **FAISS** | Meta製ライブラリ、GPU対応、数十億規模 |
| **Marqo** | マルチモーダル（テキスト+画像+動画） |
| **Turbopuffer** | マルチテナントSaaS（名前空間制限なし） |
| **Zilliz Cloud** | Milvus マネージド版、100M-1B+ |

### 選定フレームワーク

| 状況 | 推奨 |
|---|---|
| 既存 PostgreSQL (<50M) | pgvector + pgvectorscale から始める |
| 既存 Elasticsearch/MongoDB/Redis | 既存スタックにベクトル検索を追加 |
| 新規 (<10M) | Qdrant Cloud (無料枠) or ChromaDB (プロトタイプ) |
| 新規 (10-100M) | Pinecone (最も簡単) / Weaviate (ハイブリッド) / Milvus (コスト効率) |
| 新規 (>100M) | Milvus/Zilliz Cloud or Pinecone serverless |

### 注目トレンド

- **pgvectorscale** が50Mベクトルで99%リコール時にQdrantの11.4倍のQPSを達成（2025年5月ベンチマーク）。拡張型も専用DBと競合可能に
- 市場規模: 2024年 $1.73B → 2032年 $10.6B 予測
- **専用DB vs 拡張**: 50-100Mベクトル超で拡張型はスループット・レイテンシの限界に達する。それ以下なら拡張型で十分なケースが多い

## オーケストレーションフレームワーク

RAG パイプラインの構築・管理を行うライブラリ・プラットフォーム。

### 二大フレームワーク

| フレームワーク | 強み | 位置づけ |
|---|---|---|
| **LangChain** | オーケストレーション: マルチステップフロー、ツール使用、エージェント、ルーティング | アプリ配線・フロー制御 |
| **LlamaIndex** | データ: 取り込み、変換、インデックス、検索チューニング、評価 | データ接続・検索最適化 |

両者は重複が増えているが、**LlamaIndex で ingestion/indexing、LangChain + LangGraph でオーケストレーション**が実用的な組み合わせ。

### その他の注目フレームワーク

| フレームワーク | 特徴 |
|---|---|
| **DSPy** | 最低フレームワークオーバーヘッド (~3.53ms)。プロンプトの自動最適化 |
| **Haystack** | 低オーバーヘッド (~5.9ms)、最少トークン使用量 (~1.57k) |
| **RAGFlow** | 深いドキュメント理解 (PDF テーブル、レイアウト解析)。TreeRAG 搭載 |
| **Dify** | ビジュアルワークフローエディタ。コーディング不要でRAGパイプライン構築 |
| **Mem0** | メモリレイヤー特化。ユーザー/セッション/エージェントレベルの永続記憶 |

## 埋め込みモデル

テキストをベクトル表現に変換するモデル。チャンクサイズと並んで検索精度への影響が最も大きい。

| モデル | 次元数 | コスト/1Mトークン | 適用 |
|---|---|---|---|
| **OpenAI text-embedding-3-large** | 3072 (削減可) | $0.13 | 汎用、多ドキュメント種別 |
| **Cohere embed-v3** | 1024 | $0.10 | 多言語、EC |
| **nomic-embed-text (ローカル)** | 768 | $0 (計算のみ) | プライバシー重視、オンプレ |
| **Snowflake Arctic Embed** | — | OSS | 高性能 OSS 選択肢 |
